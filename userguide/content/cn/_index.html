{{< /blocks/cover >}} {{% blocks/lead color="primary" %}} KubeProber 是一个针对大规模 Kubernetes 集群设计的诊断工具，用于在 kubernetes 集群中执行诊断项以证明集群的各项功能是否正常，采用云原生的方式来编排以及运行你的诊断项，支持用户自定义诊断项，同时依托底层的集群管理架构可实现针对大规模 Kubernetes 集群的统一运维管理{{% /blocks/lead %}} {{< blocks/section color="dark" type="features">}} {{% blocks/feature icon="fab fa-soundcloud" title="云原生 !" url="https://docs.erda.cloud/1.2/manual/eco-tools/kubeprober/concepts/cluster_crd.html" %}} 核心逻辑采用 Operator 来实现，提供完整的Kubernetes API兼容性 {{% /blocks/feature %}} {{% blocks/feature icon="fas fa-th" title="大规模 !" url="https://docs.erda.cloud/1.2/manual/eco-tools/kubeprober/concepts/arch.html" %}} 采用 Master/Agent 的架构来管理海量集群，基于 WebSocket 连接理论支持的集群数量无上限 {{% /blocks/feature %}} {{% blocks/feature icon="fab fa-github" title="开源 !" url="https://github.com/erda-project/kubeprober/blob/master/CONTRIBUTING.md" %}} 遵循 Apache 2.0 许可证，如果您对本项目想做出贡献，请参考 Contributing to KubeProber。 {{% /blocks/feature %}} {{< /blocks/section >}}
